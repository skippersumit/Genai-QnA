
GENAI DOCUMENT Q&A BACKEND PROJECT – CHAT BACKUP SUMMARY
========================================================

This file is a consolidated backup of the conversation-driven development process
for building the GenAI Document Q&A backend system. It summarizes the daily work,
design decisions, architecture evolution, and backend engineering concepts learned
through the process.

Project Stack
-------------
Java 17
Spring Boot 3
Ollama / Groq LLM
Qdrant Vector DB
Redis (Caching)
Docker
PostgreSQL (metadata – optional)
Executor + Queue pattern for async ingestion

---------------------------------------------------------
DAY 1 – System Design & Scope
---------------------------------------------------------
Defined the problem: Users want to query large documents using natural language.

System goals:
• Upload documents
• Chunk text
• Generate embeddings
• Store vectors
• Query documents via semantic search
• Use LLM to generate grounded answers

Non-goals:
• No fine-tuning
• No multi-tenant auth
• No UI
• No multimodal support

High-level architecture:
Client → Spring Boot API → Embedding Service → Vector DB → LLM

DAY 16 – Rate Limiting
---------------------------------------------------------
Added protection against abuse.

Strategy:
20 requests per minute per API key.

Response on exceed:
HTTP 429 Too Many Requests

---------------------------------------------------------
DAY 17 – Redis Caching
---------------------------------------------------------
Integrated Redis caching.

Cached:
• Embeddings
• /ask responses

Impact:
Cold request: ~830 ms
Cached request: ~10–20 ms

Huge latency improvement.
